Testopstelling MS SharePoint

MS SharePoint is een platform dat op een een server draait. Je kan hier bestanden op opslaan of delen. Het is beter voor bedrijven dan Google Drive, in die zin dat het op je eigen server draait en dat je niet afhankelijk bent van Google. Aangezien alles gebeurt via het lokale netwerk van het bedrijf heb je geen internetverbinding nodig. Er wordt gebruik gemaakt van bibliotheken om de verschillende bestanden te sorteren.

Bedrijven en scholen gebruiken SharePoint voor het maken van websites. SharePoint is een veilige locatie voor het opslaan, organiseren, delen en gebruiken van informatie vanaf elk apparaat. Je hebt alleen een webbrowser nodig, zoals Internet Explorer, Chrome of Firefox. Dit wil dus zeggen dat iedereen thuis en op school in dezelfde mappenstructuur kan werken. Ook is er op MS SharePoint functionaliteit voor agenda’s fora en enquêtes.

Voordelen hiervan zijn:

  • Omdat MS SharePoint online versies bezit van Word, Excel, Powerpoint en andere programma’s (Office 365) kan je online je bestanden hierop maken en opslagen. Hierdoor moet je geen bestanden meer uploaden en downloaden via google Drive of Smartschool.

  • Het is beschikbaar op de meeste apparaten zoals tablets, smartphones, computers dus de leerlingen kunnen overal hiervan gebruik maken door op de Sharepoint website van de school in te loggen.

  • Doordat SharePoint erg gestructureerd is, is er een enorme tijdwinst bij het opzoeken van informatie.

  • Op het stagebedrijf van Bert werd SharePoint ook gebruikt om mails rond te sturen. Dit was vooral handig om ervoor te zorgen dat iedereen dezelfde templates voor hun mail gebruikte (iedereen gebruikt dezelfde online handtekening).

  • Bij SharePoint 2007 en SharePoint 2010 was het moeilijk om een website op te stellen. Maar bij SharePoint 2013 is veel moderner en de sites werken goed op kleinere schermen van tablets en smartphones.

  • SharePoint kan ook gebruikt voor het voeren van versiebeheer, één van de belangrijkste voordelen hiervan is dat je altijd kan terugvallen op een vorige versie wanneer er een fout in een bestand zit.

  • Het is heel eenvoudig om samen aan projecten te werken met partnerbedrijven, ook de communicatie hierbij verloopt vlot dankzij SharePoint.

Nadelen hiervan zijn:

  • Sharepoint zou dus Smartschool en Google Drive kunnen vervangen maar beide applicaties zijn beter in wat ze doen. Op onze Google accounts hebben wij bijvoorbeeld ongelimiteerde opslagruimte op Google Drive. Op Smartschool is er ook al gespecialiseerde functionaliteit voorzien voor schoolactiviteiten zoals de rapporten of de kalender/agenda.

  • SharePoint is best moeilijk op te zetten en draaiende te houden. Bert heeft tijdens zijn stage contact gemaakt met SharePoint en vond het redelijk overweldigend want er waren zoveel opties beschikbaar. Zijn stagementor zei ook dat het niet makkelijk te onderhouden is omdat om SharePoint heel ingewikkeld is en het vergt veel tijd om het goed voor je bedrijf te laten functioneren.

  • SharePoint is ook niet de goedkoopste tool ter wereld. De kosten voor het programma zelf zijn best nog wel goedkoop maar je moet ook nog allerlei andere voorzieningen aanschaffen. Je moet bijvoorbeeld per user 4,20 euro per maand betalen en mogelijk is het nodig nog extra hardware aan te schaffen.

  • Op SharePoint staan veel klant- en personeelsgegevens, financiële gegevens, productinformatie en patenten. Het is een groot risico dat deze in de verkeerde handen komen als werknemers niet goed omgaan met de vertrouwelijke gegevens of als de server wordt gehackt. De informatie kan afgeschermd worden met SharePoint maar de risico’s kunnen niet onderschat worden.

  • Aangezien SharePoint een platform is van Microsoft, trekt het de software van Microsoft ook wel voor. Word, Powerpoint en dergelijken werken dus heel goed met SharePoint. Maar als je een iets onbekender programma gebruikt, merk je wel eens dat SharePoint hier niet zo goed mee samenwerkt.

Dan hebben we nog een probleem in de test van CreateFreeFields opgelost, dat was de modal dialog error van gisteren. Nu zijn ook onze eerste twee testen zo goed als afgewerkt. Morgen zullen we waarschijnlijk een nieuwe opdracht krijgen van Wim of we kunnen ook aan de reguliere expressies werken want daar hadden we vandaag geen tijd voor.

Nadat we het antwoord hadden gevonden met hulp van Matthias heeft Wim ons een opdracht gegeven om de lijst van bezoekers aan te vullen, maar dan bleek dat deze al voorgemaakt was. Review of a software tool

under Engels

Review of a hardware, software or network tool in the training company

In this text I am going to review the QF-Test tool. This tool is used to test the functionality of an application. It’s especially popular among big companies, they use the tool to check if their software application is working like it’s supposed to.

One of those companies is SwissLife, they have been using the tool for four years now and they are very satisfied with it as they find the tool well structured and easy to use. I agree with Swisslife, because I think it’s a very useful tool. Unlike other test tools, QF-Test is very easy to upgrade to a better version. Another advantage of the tool is that all the different test suites are saved in small and structured files, which creates a clear view of all the work you have already done. The tool can also collaborate with Java and HTML which are both part of the most commonly used programming languages.

On the other hand, the downsides are that the testcase files are not encrypted which causes them to occupy more memory. They purposely did this so that the files would be readable. In the two weeks of experience I had with the tool I came across a few bugs. Sometimes the tool can’t recognize certain elements of the application that is getting tested. Another downside is that you can’t find much about the tool on the internet so if you get stuck on a problem it’s not that easy to find the answer. QFS, the company who owns QF-Test, found an answer to that problem and they put a tutorial online but I think this tutorial is not detailed enough and most of the time I had to keep trying new things until it finally worked. Finally it’s very important to check if your application is able to interact with QF-Test because there are applications which can’t interact with QF-Test.

I absolutely recommend the QF-Test tool to companies that want to make sure their software application is still working properly. You don’t need any education or training to use it. You just need some programming knowledge and you’re good to go. There are more tools like QF-Test on the market. But what QF-Test distinguishes from the others is the well-structured graphical user interface and the efficiency to do a lot of things in a short time.

Casestudy

under Systeembeheer

Onderwerp

QF-test is een tool om via geautomatiseerde testen, Java en web applicaties te testen. Het wordt vooral gebruikt door bedrijven om hun Software applicaties te testen op fouten. Ook mijn stagebedrijf, GET N.V., gebuikt deze tool om hun applicaties te testen. GET doet constant aanpassingen aan hun applicaties en door al deze testen elke nacht te laten lopen weten ze of alles nog goed werkt. Wij hebben ook zelf een paar fouten gevonden in PrimeTime, de applicatie van GET N.V.. Er zijn nog veel gelijkaardige tools op de makt. Maar toch hebben ze voor QF-Test gekozen omdat deze tool een GUI heeft en de testen zijn geautomatiseerd en dat bespaard veel werk.

Bevindingen

Als je er net mee begint te werken is het een beetje zoeken want je weet niet wat de functies van alle knoppen zijn. Eens je daarmee weg bent is het vrij aangenaam werken aangezien de tool voorzien is van een gebruiksvriendelijke GUI, is het meeste werk gebeurd met enkele klikken van de muis. Maar het was niet altijd zo simpel, soms kwamen we problemen tegen die niet opgelost konden worden met de door de tool voorziene mogelijkheden. De enige oplossing was dan om zelf een script te schrijven dat dit wel kon. Aangezien deze scriptjes in groovy moesten geschreven worden was dat niet altijd even makkelijk voor ons want we hadden voorheen we daar aan de slag gingen geen enkele kennis van deze programmeertaal.

Eigen ondervinding

Ik persoonlijk vind het een heel aangename tool om mee te werken. Zoals reeds gezegd is het in het begin een beetje knoeien maar na twee dagen ben je er zeker en vast mee weg. Er zijn ook een paar handige functies voorzien zoals bijvoorbeeld loop-functies. Zo kan je herhaaldelijk dezelfde acties uitvoeren en dat bespaard veel werk. Een nadeel van de tool is dat de scriptjes in groovy of jython geschreven moeten worden. Dat zijn programmeertalen die beginnende programmeurs hoogstwaarschijnlijk nog niet kennen en dat kan je al wel eens vertragen bij je werk.
